# 📦 仓库卡片线框示例（`order-service`）

```
┌───────────────────────────────────────────────┐
│ order-service                      ✅ 成功     │  ← 仓库名 + 状态
│-----------------------------------------------│
│ 分支：feature/login-fix                       │
│ 环境URL：https://order-service-login-fix.stg.g123.jp │
│ 最新部署时间：2025-09-12 14:32                │
│ 部署人：张三                                  │
│-----------------------------------------------│
│ [停用]   [重新部署]   [查看日志]               │  ← 常用操作按钮
└───────────────────────────────────────────────┘
```

---

# 🎨 设计要点

1. **仓库名 + 状态**

   * 仓库名加粗，左对齐。
   * 状态在右上角，用颜色+图标（✅ 成功 / ❌ 失败 / ⏳ 部署中）。

2. **核心信息**（中部）

   * 只展示当前有效分支、环境 URL、最新部署时间、部署人。
   * 避免冗余，把重点信息一眼看到。

3. **操作区**（底部）

   * 提供最常用的 3 个操作按钮：停用  / 重新部署/ 查看日志。
   * 其他不常用操作放到详情页，不在卡片里堆叠。

4. **布局**

   * 网格布局：一行 2–3 个卡片。
   * 卡片 hover 时加阴影，整个卡片可点击进入「详情」，但底部按钮可以直接触发操作。

---

📌 这样，用户进入「项目下的仓库列表」时：

* 看到的就是一排排卡片（每个仓库一张卡），每张卡上都清晰展示：**当前有效分支 + 部署结果**。
* 无需点详情页，就能快速判断哪个仓库有问题，直接点「查看日志」排查。

# 🎨 设计要点

1. **顶部卡片**（和列表页风格统一）：

   * 仓库名 + 当前有效分支状态放在最显眼的位置。
   * 提供一键操作（重新部署、查看日志、停用环境、删除）。

2. **分区展示**：

   * **基本信息** → 只看仓库的元信息。
   * **部署记录** → 默认展开最近 N 条，状态用颜色+图标。
   * **分支列表** → 显示全部分支，方便查看，但只在详情里出现，不占列表空间。

3. **操作快捷化**：

   * 部署记录里直接加「查看日志」按钮。
   * 分支列表可以「手动触发部署」。

---

📌 整体效果：

* 列表页 → 卡片布局，快速扫全局。
* 详情页 → 卡片 + 分区信息，操作一目了然。
* 日志页 → 深入具体部署，方便排查。
