多环境整理业务逻辑改造

1. 进入多环境菜单菜单，是项目列表，展示信息为 项目名称、项目描述、项目最新一次提交的pr的分支和commit 信息、提交人
2. 点击项目进入，参考vercel 的实现（看我的截图），项目下有两个tab，部署列表和设置，列表展示 仓库名称、commit ID、部署结果、部署分支、commit 信息、提交人、状态（部署完成、测试完成）、部署日期、操作（测试完成、销毁环境）
3. 点击部署列表详情，可查看部署日志
4. 点击设置，参考vercel 的实现（看我的截图）,
   1. 基本信息：可配置项目名称、项目关联的github仓库，可执行删除操作，删除需要二次确认
   2. 环境参数：可配置生效的 yaml 文件，每个仓库的配置可以不一样，需要单独为仓库配置，未配置则菜单标红
